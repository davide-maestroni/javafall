Parallel programming on the go.

Overview

Android integration of RxJava based on the jroutine-android library.

This module provides classes integrating Android routines with the RxJava library.
RxJava data manipulation is executed inside dedicated Android Loaders.

Main classes

  • JRoutineRx

    Utility class enabling an observable to dispatch data to a dedicated Loader.

  • JRoutineRxCompat

    Utility class enabling an observable to dispatch data to a dedicated Loader.
    This class is specifically meant to employ the Android Support library for retro-compatibility.

Dependencies

Below a full list of the included modules:

  • jroutine-android-core — core routine and channel classes
  • jroutine-core — core routine and channel classes

Usage examples

The code below shows how it is possible to configure an observable, so to employ Loaders to execute data processing.

Example 1: via Loader.

The lambda notation used in the example is just for demonstration purpose. 

    
        public class MainActivity extends Activity {

            private static Action1<String> getAction(final Context context) {
                return new Action1<String>() {

                    @Override
                    public void call(final String s) {
                        Toast.makeText(context, s, Toast.LENGTH_LONG).show();
                    }
                };
            }

            private static Func1<String, String> getMappingFunction() {
                return new Func1<String, String>() {

                    @Override
                    public String call(final String s) {
                        return s + "!";
                    }
                };
            }

            @Override
            protected void onCreate(final Bundle savedInstanceState) {
                super.onCreate(savedInstanceState);
                setContentView(R.layout.main_layout);
                JRoutineRx.withObservable(Observable.just("Hello RxJava"))
                          .applyLoaderConfiguration()
                          .withInvocationId(R.id.hello_invocation)
                          .configured()
                          .subscribeOn(loaderFrom(this))
                          .map(getMappingFunction())
                          .observeOn(AndroidSchedulers.mainThread())
                          .subscribe(getAction(this));
            }
        }
