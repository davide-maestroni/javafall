Parallel programming on the go.

Overview

Android integration of RxJava2 based on the jroutine-android-core library.

This module provides classes integrating Android routines with the RxJava library.

Main classes

  • JRoutineLoaderFlowable

    Utility class enabling an Flowable to dispatch data to a dedicated Loader.

  • JRoutineLoaderFlowableCompat

    Utility class enabling an Flowable to dispatch data to a dedicated Loader.
    This class is specifically meant to employ the Android Support library for retro-compatibility.

Dependencies

Below a full list of the included modules:

  • jroutine-android-core — core routine and channel classes
  • jroutine-core — core routine and channel classes
  • jroutine-rx — RxJava integration

Usage examples

The code below shows how it is possible to configure an Flowable, so to employ Loaders to execute data processing.

Example 1: via Loader.

The lambda notation used in the example is just for demonstration purpose. 

    
        public class MainActivity extends Activity {

            private static Consumer<String> getConsumer(final Context context) {
                return new Consumer<String>() {

                    @Override
                    public void accept(final String s) {
                        Toast.makeText(context, s, Toast.LENGTH_LONG).show();
                    }
                };
            }

            private static Function<String, String> getMappingFunction() {
                return new Function<String, String>() {

                    @Override
                    public String apply(final String s) {
                        return s + "!";
                    }
                };
            }

            @Override
            protected void onCreate(final Bundle savedInstanceState) {
                super.onCreate(savedInstanceState);
                setContentView(R.layout.main_layout);
                JRoutineLoaderFlowable.with(Flowable.just("Hello RxJava").map(getMappingFunction()))
                                      .loaderConfiguration()
                                      .withInvocationId(R.id.hello_invocation)
                                      .apply()
                                      .observeOn(loaderFrom(this))
                                      .subscribe(getConsumer(this));
            }
        }
