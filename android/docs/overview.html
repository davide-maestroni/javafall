Parallel programming on the go.

Overview

Android platform customization of the jroutine-lib library.

The module collects all the main features by including the base modules and exposing them trough a single entry point class.
It is anyway possible to include just the needed sub-modules as dependencies.

Main classes

  • JRoutineAndroid

    Utility class exposing the JRoutine Android modules functionalities.

  • JRoutineAndroidCompat

    Utility class exposing the JRoutine Android modules functionalities.
    This class is specifically meant to employ the Android Support library for retro-compatibility.

Dependencies

Below a full list of the included modules:

  • jroutine-android-channel — utility classes extending channel functionalities
  • jroutine-android-core — core routine and channel classes
  • jroutine-android-method — routine implementation through a class method
  • jroutine-android-proxy — code generation of routines wrapping object and class methods
  • jroutine-android-reflect — routine wrapping object and class methods
  • jroutine-android-stream — stream-like routine concatenation
  • jroutine-channel — utility classes extending channel functionalities
  • jroutine-core — core routine and channel classes
  • jroutine-function — utility classes enabling function-based routines
  • jroutine-lib — core classes facade
  • jroutine-reflect — routines wrapping object and class methods
  • jroutine-stream — stream-like routine concatenation
