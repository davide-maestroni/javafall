Parallel programming on the go.

Overview

Java annotation proxy code generator based on the JRoutine-Android libraries.

This module provides classes to enable asynchronous invocations of object or class methods, by wrapping them in compile time generated routines.
Methods are invoked via generated code inside dedicated Android Services and Loaders.

Main concepts

  • LoaderProxy

    A LoaderProxy annotation will indicate which interface has to be implemented by the generated code to proxy the target class or instance, so that its methods are executed inside a Loader instance.

  • LoaderProxyCompat

    A LoaderProxyCompat annotation will indicate which interface has to be implemented by the generated code to proxy the target class or instance, so that its methods are executed inside a Loader instance. This class is specifically meant to employ the Android Support library for retro-compatibility.

  • ServiceProxy

    A ServiceProxy annotation will indicate which interface has to be implemented by the generated code to proxy the target class or instance, so that its methods are executed inside a dedicated Service.

  • LoaderProxyRoutineBuilder

    Each generated proxy instance needs an invocation target and a specific configuration, so proxy routine builders provide a convenient way to set the required options. Instances of this builder will handle only interfaces annotated with LoaderProxy or LoaderProxyCompat annotations. By design builders are not synchronized since the same instance is not meant to be used from different threads.

  • ServiceProxyRoutineBuilder

    Each generated proxy instance needs an invocation target and a specific configuration, so proxy routine builders provide a convenient way to set the required options. Instances of this builder will handle only interfaces annotated with ServiceProxy annotations. By design builders are not synchronized since the same instance is not meant to be used from different threads.

Main classes

  • JRoutineLoaderProxy

    Utility class acting as a factory of Loader proxy routine builders.

  • JRoutineLoaderProxyCompat

    Utility class acting as a factory of Loader proxy routine builders.

  • JRoutineServiceProxy

    Utility class acting as a factory of Service proxy routine builders.

Dependencies

Below a full list of the included modules:

  • JRoutine-AndroidCore: core routine and channel classes
  • JRoutine-AndroidObject: routine wrapping object and class methods
  • JRoutine-AndroidProcessor: routine annotation pre-processor
  • JRoutine-Core: core routine and channel classes
  • JRoutine-Object: routines wrapping object and class methods
  • JRoutine-Processor: routine annotation pre-processor
  • JRoutine-Proxy: code generation of routines wrapping object and class methods

Usage examples

The code below shows how it is possible to load an image from the network in a background thread, by letting the library automatically handle any change in the configuration.
Note that the very same examples can be re-written using JRoutineLoaderProxyCompat and a FragmentActivity, or the JRoutineServiceProxy class, in order to make the processing happen in a Service instead of a Loader.

Example 1: via code generation.

Note that this approach is fairly naive, since it lets the framework retain the instance of the downloaded bitmap.
A smarter implementation would involve the use of a dedicated cache. 

    
        public class MainActivity extends Activity {

            private static final String IMAGE_URI = "http://...";

            private static final String LOAD_BITMAP = "load_bitmap";

            @Override
            protected void onCreate(final Bundle savedInstanceState) {
                super.onCreate(savedInstanceState);
                setContentView(R.layout.main_layout);
                final ImageView imageView = (ImageView) findViewById(R.id.image);
                JRoutineLoaderProxy.on(loaderFrom(this))
                                   .with(instanceOf(BitmapLoader.class))
                                   .buildProxy(BitmapLoaderItf.class)
                                   .loadFromUri(URI.create(IMAGE_URI))
                                   .bind(new TemplateChannelConsumer<Bitmap>() {

                                       @Override
                                       public void onError(@NotNull final RoutineException error) {
                                           final Throwable cause = error.getCause();
                                           Toast.makeText(MainTestActivity.this, (cause != null) ?
                                                   cause.getMessage() : "Cannot load image",
                                                   Toast.LENGTH_LONG).show();
                                       }

                                       @Override
                                       public void onOutput(final Bitmap bitmap) {
                                           imageView.setImageBitmap(bitmap);
                                       }
                                   });
            }

            @LoaderProxy(BitmapLoader.class)
            public interface BitmapLoaderItf {

                @Alias(LOAD_BITMAP)
                @CacheStrategy(CacheStrategyType.CACHE_IF_SUCCESS)
                @AsyncOutput
                @NotNull
                Channel<?, Bitmap> loadFromUri(@NotNull URI uri);
            }

            public static class BitmapLoader {

                @Alias(LOAD_BITMAP)
                @NotNull
                public Bitmap loadFromUri(@NotNull final URI uri) throws IOException {
                    final URLConnection urlConnection = uri.toURL().openConnection();
                    final InputStream inputStream = urlConnection.getInputStream();
                    try {
                        final Bitmap bitmap = BitmapFactory.decodeStream(inputStream);
                        if (bitmap == null) {
                            throw new IOException("failed to decode bitmap");
                        }

                        return bitmap;

                    } finally {
                        inputStream.close();
                    }
                }
            }
        }
Example 2: via code generation with custom Context.

It is possible to customize the creation of the wrapped object by using a Context wrapper as shown below. 

    
        public class MainActivity extends Activity {

            private static final String IMAGE_URI = "http://...";

            private static final String LOAD_BITMAP = "load_bitmap";

            @Override
            protected void onCreate(final Bundle savedInstanceState) {
                super.onCreate(savedInstanceState);
                setContentView(R.layout.main_layout);
                final ImageView imageView = (ImageView) findViewById(R.id.image);
                JRoutineLoaderProxy.on(loaderFrom(this, new MyContext(this)))
                                   .with(instanceOf(BitmapLoader.class))
                                   .buildProxy(BitmapLoaderItf.class)
                                   .loadFromUri(URI.create(IMAGE_URI))
                                   .bind(new TemplateChannelConsumer<Bitmap>() {

                                       @Override
                                       public void onError(@NotNull final RoutineException error) {
                                           final Throwable cause = error.getCause();
                                           Toast.makeText(MainTestActivity.this, (cause != null) ?
                                                   cause.getMessage() : "Cannot load image",
                                                   Toast.LENGTH_LONG).show();
                                       }

                                       @Override
                                       public void onOutput(final Bitmap bitmap) {
                                           imageView.setImageBitmap(bitmap);
                                       }
                                   });
            }

            @LoaderProxy(BitmapLoader.class)
            public interface BitmapLoaderItf {

                @Alias(LOAD_BITMAP)
                @CacheStrategy(CacheStrategyType.CACHE_IF_SUCCESS)
                @AsyncOutput
                @NotNull
                Channel<?, Bitmap> loadFromUri(@NotNull URI uri);
            }

            public static class BitmapLoader {

                @Alias(LOAD_BITMAP)
                @NotNull
                public Bitmap loadFromUri(@NotNull final URI uri) throws IOException {
                    final URLConnection urlConnection = uri.toURL().openConnection();
                    final InputStream inputStream = urlConnection.getInputStream();

                        if (bitmap == null) {
                            throw new IOException("failed to decode bitmap");
                        }

                        return bitmap;

                    } finally {
                        inputStream.close();
                    }
                }
            }

            private static class MyContext extends FactoryContextWrapper {

                private final BitmapLoader mLoader;

                public MyContext(@NotNull final Context base) {
                    super(base);
                    mLoader = new BitmapLoader();
                }

                @Nullable
                public <TYPE> TYPE geInstance(@NotNull final Class<? extends TYPE> type,
                        @NotNull final Object... args) {
                    if (type == BitmapLoader.class) {
                        return type.cast(mLoader);
                    }

                    return null;
                }
            }
        }
Example 3: via code generation avoiding reflection.
